1. [CleanArchitecture (Jason Taylor)](https://github.com/jasontaylordev/CleanArchitecture)
2. [eShopOnWeb](https://github.com/dotnet-architecture/eShopOnWeb)

4. [hoc081098/netauth-ddd-cqrs-clean](https://github.com/hoc081098/netauth-ddd-cqrs-clean)

6. [banking-kata-dotnet](https://github.com/valentinajemuovic/banking-kata-dotnet)

## O que observar nas referencias

- Organizacao de testes por camada/modulo.
- Uso de doubles/mocks de forma equilibrada.
- Casos de teste de regras de negocio e bordas.
- Testes de arquitetura para evitar acoplamento indevido.
- Estrategia de CI para rodar testes em PR.

## Playbook TDD para o Time

- Fatia pequena de requisito.
- Teste falhando primeiro.
- Implementacao minima.
- Refatoracao guiada por testes.
- Commit pequeno e frequente.
